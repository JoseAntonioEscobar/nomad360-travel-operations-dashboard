Expedition Readiness ScoreMeasure
VAR _serviceScore = CALCULATE(DIVIDE(COUNTROWS(FILTER(FactPartnerServices, [Status] = "Confirmed")),COUNTROWS(FactPartnerServices))) * 100 VAR _flightScore = CALCULATE(DIVIDE(COUNTROWS(FILTER(DimTravelers, [FlightStatus] = "Confirmed")),COUNTROWS(DimTravelers))) * 100 VAR _paymentScore = DIVIDE([TotalPaid], [TotalDue]) * 100 VAR _docScore = CALCULATE(DIVIDE(COUNTROWS(FILTER(FactDocuments, [Complete] = TRUE())),COUNTROWS(FactDocuments))) * 100 RETURN _serviceScore * 0.30 + _flightScore * 0.20 + _paymentScore * 0.20 + _docScore * 0.15 + (DIVIDE([RegisteredTravelers],[ExpectedTravelers]) * 100) * 0.15

SWITCH(TRUE(),[Expedition Readiness Score] >= 90, "Ready",[Expedition Readiness Score] >= 70, "At Risk","Critical")

VAR _totalDue = SUM(FactTripParticipants[AmountDue]) VAR _totalPaid = SUM(FactTripParticipants[AmountPaid]) RETURN _totalDue - _totalPaid

CALCULATE(SUMX(DimTrips, DimTrips[TotalValue]),DimTrips[Status] IN { "Active", "Upcoming" })

Design Philosophy

Why This Dashboard Works for Expedition Operations

This platform was designed around a fundamental insight: expedition operators don't manage trips in isolation — they orchestrate dozens of concurrent journeys across multiple countries, each with its own supply chain, regulatory requirements, and traveler mix.

The Readiness Score is the north star metric because it distills operational complexity into a single, actionable number. Unlike generic project management scores, this one is weighted by what actually matters in travel operations: service confirmations carry 30% weight because a trip cannot run without confirmed hotels, guides, and transport.

  • Every visual supports a specific operational decision, not just information display
  • Color follows traffic-light semantics that operations teams already use instinctively
  • The departure countdown creates urgency without alarm — “6 trips depart within 7 days” tells you exactly where to focus today
  • Financial metrics are embedded alongside operational ones because cash flow drives service confirmation capacity
  • The network diagram shows operational relationships that spreadsheets cannot — seeing which hubs connect to which destinations triggers proactive supplier outreach
